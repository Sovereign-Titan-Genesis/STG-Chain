# STG-Chain Governance Model (Draft)

## Proposal Lifecycle
1. Submit proposal (syarat reputasi/stake)
2. Deposit STG untuk anti-spam
3. Voting 7–14 hari, quorum 40%
4. Execution otomatis jika lolos

## Voting Power
Voting power = stake weight + reputasi weight
VP = α * (stake/total_stake) + β * (reputation/max_reputation)

## Emergency Governance
- Emergency pause oleh mayoritas validator
- Fast track proposal untuk upgrade keamanan

## Transparency & Audit
- Semua proposal, voting, hasil dicatat on-chain
- Audit trail untuk setiap keputusan governance
